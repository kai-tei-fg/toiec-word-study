# えいたんごクイズ — アプリケーション仕様書

## 1. 概要

子供が英単語を覚えているかどうかをチェックするWebアプリケーション。  
親が単語帳を管理し、子供がスマホでクイズに挑戦する運用を想定している。

---

## 2. フォルダ構成

```
📁 プロジェクトルート/
├── vocab-quiz.html            # 子供向け：クイズアプリ本体
├── worklist-manager.html      # 親向け：単語帳管理ツール
├── sample_words.csv           # サンプルCSVファイル
└── 📁 worklists/
    └── worklists.js           # 単語帳データ（管理ツールで生成）
```

## 3. 技術アーキテクチャ

| 項目 | 内容 |
|------|------|
| フロントエンド | HTML + CSS + JavaScript（単一ファイル構成） |
| サーバー | 不要（ローカルで `file://` でも動作可能） |
| 外部依存 | Google Fonts（M PLUS Rounded 1c, Nunito） |
| ブラウザAPI | Web Speech API（発音）、Web Share API（シェア）、FileReader API |
| データ連携 | `worklists/worklists.js` を `<script src>` で自動読み込み |

## 4. CSVファイルフォーマット

### 4.1 形式

```csv
単語,意味
apple,りんご
book,本
cat,ねこ
```

- 1行目のヘッダー（`単語,意味` または `word,意味`）は任意（あっても無くてもOK）
- 区切り文字: カンマ(`,`) またはタブ(`\t`)
- エンコーディング: UTF-8（BOM付き可）、Shift_JIS にも自動対応
- 文字化け検出機能あり（UTF-8 → Shift_JIS の自動フォールバック）

### 4.2 読み込み処理

1. `FileReader.readAsArrayBuffer()` でバイナリ読み込み
2. `TextDecoder('UTF-8')` でデコード試行
3. BOM（`0xFEFF`）があれば除去
4. パース実行、文字化け（`\uFFFD`）検出時は `TextDecoder('Shift_JIS')` で再試行

## 5. worklists.js フォーマット

### 5.1 データ構造

```javascript
// えいたんごクイズ - 単語帳データ
// Generated: 2026/3/20 12:00:00
window.WORKLISTS = {
  "単語帳名1": [
    {word: "apple", meaning: "りんご"},
    {word: "book", meaning: "本"}
  ],
  "単語帳名2": [
    {word: "elephant", meaning: "ぞう"},
    {word: "giraffe", meaning: "きりん"}
  ]
};
```

- `window.WORKLISTS` にオブジェクトを代入
- キー: 単語帳名（文字列）
- 値: `{word: string, meaning: string}` の配列

### 5.2 読み込み方法

両HTML（クイズアプリ・管理ツール）とも `<script src="worklists/worklists.js">` で自動読み込み。  
ファイルが存在しない場合は `onerror` でエラーを吸収し、CSVアップロードのみで動作する。

## 6. クイズアプリ（vocab-quiz.html）

### 6.1 対象ユーザー

子供（主にスマホ操作）

### 6.2 画面遷移

```
[HOME画面] → [QUIZ画面] → [RESULTS画面]
              ↑                    |
              └────── もういちど ───┘
```

### 6.3 HOME画面

#### 6.3.1 単語帳選択エリア

- `worklists/worklists.js` から自動読み込みした単語帳をチェックボックスリストで表示
- **複数選択可能**
- 各項目に単語帳名と語数を表示
- `worklists.js` が存在しない場合はこのセクション非表示

#### 6.3.2 CSVアップロードエリア

- ドラッグ＆ドロップまたはタップでCSVファイルをアップロード
- **複数ファイル対応**（`multiple` 属性）
- アップロード済みファイルはタグ表示（✕ボタンで個別削除可能）
- 単語帳選択との**併用可能**（両方の単語が合算される）

#### 6.3.3 単語合算と重複排除

- 選択した単語帳 + アップロードCSVの全単語を合算
- **英単語（小文字化）で重複排除**（`Map` で管理）
- 合算後の総単語数を表示

#### 6.3.4 テスト数選択

合計単語数 > 0 の場合に表示：

| 選択肢 | 説明 | デフォルト |
|--------|------|-----------|
| ぜんぶ | 全単語を対象 | ✅ デフォルト選択 |
| いくつかだけ | 指定数をランダム抽出 | 未選択 |

- 「いくつかだけ」選択時、数値入力欄を表示
  - デフォルト値: `min(50, 総単語数)`
  - 範囲: `1 〜 総単語数`
  - バリデーションあり

#### 6.3.5 テスト開始ボタン

- 「🚀 テストをはじめる！」

### 6.4 QUIZ画面

#### 6.4.1 進捗バー

- 「もんだい X / Y」テキスト + パーセント表示
- グラデーション付きプログレスバー（黄→オレンジ）

#### 6.4.2 出題ルール

- 各単語に対して **2種類の問題** を生成
  - **選択式（choice）**: 英単語を表示 → 正しい日本語を4択から選ぶ
  - **入力式（typing）**: 日本語の意味を表示 → 英単語をタイピング入力
- 全問題をシャッフルしてランダム出題
- 合計問題数 = 選択した単語数 × 2

#### 6.4.3 選択式問題

- バッジ: 「🔤 えらんでね」（ピンク系）
- 英単語を大きく表示（Nunito フォント, 2.6rem）
- 🔊 発音ボタン（英単語の横に配置）
- 4つの選択肢を縦1列で表示（A/B/C/D ラベル付き）
- 選択肢の生成: 正解1 + 他の単語の意味からランダム3つ
- 回答後: 正解は緑、不正解は赤にハイライト

#### 6.4.4 入力式問題

- バッジ: 「✏️ にゅうりょくしてね」（紫系）
- 日本語の意味を大きく表示（2rem, 紫色）
- テキスト入力欄 + 「こたえる」ボタン
- Enter キーでも回答可能
- 判定: **大文字・小文字を区別しない**（`.toLowerCase()` 比較）

#### 6.4.5 フィードバック

| 正解 | 不正解 |
|------|--------|
| 🎉 せいかい！（緑背景） | 😢 ざんねん…（赤背景） |
| 詳細なし | 「こたえ: {正解}」を表示 |

- 入力式で正解した場合、自動で英単語を発音
- フィードバック後「つぎへ ➡️」ボタン表示

#### 6.4.6 発音機能

- **Web Speech API** (`SpeechSynthesisUtterance`) 使用
- 言語: `en-US`（アメリカ英語）
- 音声選択の優先順位:
  1. 名前で優先: Samantha, Karen, Victoria, Tessa, Allison, Ava, Google US English, Google UK English Female, Microsoft Zira, Microsoft Jenny, Microsoft Aria, Daniel, Alex
  2. フォールバック: `en-US` → `en-GB` → 任意の英語音声
- **`/^en[-_]/i` で英語音声のみフィルタ**（日本語音声を完全除外）
- パラメータ: `rate: 0.9`, `pitch: 1.1`, `volume: 0.9`
- `utter.voice` 設定時は `utter.lang` も音声の `lang` に合わせる

### 6.5 RESULTS画面

#### 6.5.1 スコア表示

- 円形スコア表示（150px）
  - 80点以上: 緑グラデーション（excellent）
  - 50〜79点: 黄→オレンジグラデーション（good）
  - 50点未満: 赤グラデーション（needs-work）
- メッセージ:
  - 80+: 「🎊 すごい！よくできました！」
  - 50〜79: 「👍 いいかんじ！」
  - 〜49: 「💪 つぎはもっとできるよ！」

#### 6.5.2 統計

- せいかい数 / まちがい数 / もんだい数（3つのボックス）

#### 6.5.3 間違えた単語一覧

- テーブル形式: 英単語 | いみ | こたえ（ユーザーの回答）
- **同じ単語の重複は排除**（`Map` で最初の間違いのみ保持）

#### 6.5.4 アクションボタン

| ボタン | 動作 | 表示条件 |
|--------|------|----------|
| 📤 シェアする | 間違えた単語をテキストでシェア | 間違いがある時 |
| 📥 CSVダウンロード | 間違えた単語をCSVでダウンロード | 間違いがある時 |
| 🔄 もういちど | HOME画面に戻る | 常に表示 |

#### 6.5.5 シェア機能

- **Web Share API** 対応端末（スマホ等）: ネイティブ共有メニュー（LINE, メール等）
- 非対応端末: **クリップボードにコピー** + トースト通知
- クリップボードも失敗した場合: `prompt()` でテキスト表示

シェアテキストフォーマット:
```
📝 えいたんごクイズの結果
スコア: 85点（17/20問 せいかい）

❌ まちがえた単語（3語）:
・apple → りんご
・book → 本
・cat → ねこ
```

#### 6.5.6 CSVダウンロード

- フォーマット: 入力CSVと同じ形式（`単語,意味`、ヘッダー付き）
- ファイル名: `まちがえた単語.csv`
- エンコーディング: UTF-8 BOM付き
- 間違えた単語のみ（重複排除済み）

## 7. 単語帳管理ツール（worklist-manager.html）

### 7.1 対象ユーザー

親（PC操作を想定、スマホでも利用可能）

### 7.2 画面構成（単一画面、3ステップ構成）

#### Step 1: 登録ずみの単語帳

- **自動読み込み**: `<script src="worklists/worklists.js">` で起動時に自動読み込み
- 読み込み成功時: 「✅ worklists/worklists.js を自動読み込みしました（X冊・Y語）」表示
- 読み込み失敗時: 手動アップロードUIをフォールバック表示
- 手動アップロード: `.js` ファイルを受け付け、`new Function()` で `WORKLISTS` を抽出

##### 登録済み単語帳の管理（アコーディオンカード形式）

各単語帳カード:
- **ヘッダー**: 📗アイコン + 単語帳名 + 語数 + アクションボタン + ▶シェブロン
- **タップで展開**: 全単語テーブル表示（英単語 | 意味）、max-height: 350pxでスクロール
- **📥 CSVダウンロード**: その単語帳のみを `単語帳名.csv` としてダウンロード
- **🗑 削除**: 確認モーダル表示 → 承認で削除

##### 削除確認モーダル

- タイトル: 「単語帳を削除」
- メッセージ: 「「{名前}」（{N}語）を削除しますか？」
- ボタン: キャンセル / 削除する
- オーバーレイクリックで閉じる

#### Step 2: あたらしい単語帳を追加

- CSVファイルアップロード（複数可）
- アップロードされた各ファイル:
  - 名前入力欄（デフォルト: ファイル名から `.csv` を除去）
  - 元ファイル名 + 語数表示
  - ✕ボタンで取り消し

#### Step 3: 保存する

- サマリー表示: 「合計 X冊（Y語）を保存します」（既存N冊 + 新規M冊 の内訳）
- 「👁 プレビュー」ボタン: 生成されるJSコードをテキスト表示
- 「💾 worklists.js をダウンロード」ボタン

##### 保存処理

1. `existing`（既存）+ `newFiles`（新規）をマージ
2. 同名の単語帳がある場合、新規が既存を上書き
3. `worklists.js` を生成してダウンロード
4. ダウンロード後、新規分を `existing` に統合し、`newFiles` をクリア
5. 画面を再レンダリング

## 9. 運用フロー

### 初回セットアップ

```
1. 親: CSVファイルを用意（単語,意味 の2列）
2. 親: worklist-manager.html を開く
3. 親: CSVをアップロード → 単語帳名を設定
4. 親: worklists.js をダウンロード → worklists/ フォルダに配置
5. 子供: vocab-quiz.html をスマホで開いてクイズ開始
```

### 日常運用

```
1. 子供: vocab-quiz.html で単語帳を選んでクイズ実施
2. 子供: 結果画面で「📤 シェアする」→ LINE等で親に送信
3. 親: 間違えた単語リストを確認
4. 親: 必要に応じて worklist-manager.html で単語帳を更新
5. 親: worklists.js をダウンロードして上書き保存
```

### 単語帳の追加（既存保持）

```
1. 親: worklist-manager.html を開く（既存データ自動読み込み）
2. 親: 新しいCSVをアップロード
3. 親: worklists.js をダウンロードして上書き保存
※ 既存の単語帳は自動で引き継がれる
```

## 10. 既知の制約事項

| 制約 | 詳細 |
|------|------|
| 発音機能 | ブラウザ/OSの音声エンジンに依存。一部環境では利用不可 |
| `file://` での制限 | CSVのフォルダ自動読み込み不可。`<script src>` は動作する場合あり |
| 選択肢の数 | 単語が4語未満の場合、4択にならない（存在する分だけ表示） |
| データ永続化 | ブラウザにデータを保存しない。全データは `worklists.js` とCSVで管理 |
| 同時編集 | 管理ツールでの同時編集は非対応（最後にダウンロードしたものが有効） |
| Web Share API | iOS Safari / Android Chrome で動作。PC Chromeでは非対応（クリップボードフォールバック） |
